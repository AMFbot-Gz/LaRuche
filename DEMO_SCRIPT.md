# Chimera — Demo Script (3 minutes)

> Audience: developers discovering Chimera for the first time (GitHub, HN, Product Hunt).
> Goal: show the full loop — boot, generate code, execute, persist — in 3 minutes flat.
> Setup before recording: `make setup` already run, Ollama already running, browser on http://localhost:3001.

---

## 00:00 — 00:30 · Boot Chimera

**Terminal action:**

```bash
make dev
```

**What you show:**
- The terminal starts Queen Node.js on `:3000`, then all 9 Python agents on `:8001`–`:8007`, then the Next.js Dashboard on `:3001`.
- Each agent logs `[READY]` as it comes online.
- Cut to browser: open `http://localhost:3001`.

**What to say:**
> "One command. Queen orchestrator, 9 agents, real-time dashboard — all local, no cloud."

**What the audience sees:**
- HiveStatus widget: all 9 agents green.
- LiveLogs widget: event stream already ticking (heartbeat pings from HealthMonitor).
- QuickAction widget: input box ready.

---

## 00:30 — 01:30 · Auto-Coder Bee — Generate & Execute a Python skill

**Browser action** (QuickAction widget):

Type and submit:
```
Count all .py files modified in the last 7 days, sorted by size descending
```

**What you show:**
- LiveLogs immediately shows events flowing: `task.received → brain.think → evolution.generate → sandbox.validate → executor.run → skill.saved`
- Cut to terminal: the Auto-Coder Bee logs the generated code snippet (a few lines of Python).
- Cut back to browser: the result appears in LiveLogs — a formatted list of files with sizes.
- The QuickAction widget shows `Skill saved: count_py_files_by_size`.

**What to say:**
> "Chimera received the description, routed it to the Brain for planning, generated Python code via Ollama locally, validated every import and AST node through the sandbox whitelist — then executed it in an isolated subprocess. The result is live in under 10 seconds."

**Callout on screen (overlay text):**
```
No code left your machine.
No API called. Pure Ollama.
```

---

## 01:30 — 02:30 · Persistent Skills — Memory across reboots

**Terminal action** (new tab, don't stop the running stack):

```bash
ls skills/generated/
```

**What you show:**
- The file `count_py_files_by_size.py` is there — created by the previous run.
- Open the file briefly to show it's real, clean Python code generated by the LLM.

**Browser action** — second task, QuickAction widget:

Type and submit:
```
Replay skill: count_py_files_by_size
```

**What you show:**
- The skill executes again instantly — no LLM call this time, it runs the cached version.
- LiveLogs shows `skill.replay → executor.run → result`.

**What to say:**
> "Every successful execution is saved as a reusable skill. Next run is instant — zero LLM inference needed. Chimera gets smarter with every task."

**Terminal action** — show the memory agent remembers context:

```bash
curl -s http://localhost:8006/memories/search \
  -H "Content-Type: application/json" \
  -d '{"query": "count python files", "n_results": 1}' | python3 -m json.tool
```

**What you show:**
- The Memory agent (ChromaDB) returns the stored context of the previous task — description, result summary, timestamp.

**What to say:**
> "The Memory agent stores every task in a vector database. Your OS remembers what it learned."

---

## 02:30 — 03:00 · Wrap — The full picture

**Browser action:** zoom out to show the full dashboard.

**What you show:**
- HiveStatus: all agents still green after 3 minutes of work.
- LiveLogs: the full event trace of both tasks, scrollable.
- Point to the QuickAction box: "This is your interface to the whole system."

**What to say:**

## Recording checklist

- [ ] Terminal font size ≥ 18pt, dark theme
- [ ] Browser zoom at 110% for readability
- [ ] Hide browser bookmarks bar
- [ ] Kill notifications (macOS Focus mode on)
- [ ] Pre-warm Ollama: `ollama run qwen3-coder "hello"` before recording to avoid cold-start lag
- [ ] Resolution: 1920×1080 minimum
- [ ] Keep the terminal and browser side by side (split screen) for maximum impact
