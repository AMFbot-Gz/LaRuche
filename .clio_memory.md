## 1. VISION & MISSION

- **Vision :** Donner à chaque développeur un système cognitif autonome — un OS qui pense, code, exécute et s'améliore à sa place, 100% local, 100% privé.
- **Mission :** Chimera est l'infrastructure d'autonomie pour les builders solo. Un monorepo cognitif (Queen Node.js + 7 agents Python + sandbox sécurisé + dashboard temps réel) qui transforme une description en langage naturel en code exécuté, validé et sauvegardé.
- **UVP (Unique Value Proposition) :** Le seul OS agentique qui fonctionne entièrement en local (Ollama), génère du code sécurisé par sandbox AST + liste blanche, et s'auto-améliore via un pipeline d'évolution intégré — sans aucune dépendance cloud obligatoire.

- **Statut:** DONE ✅

- **Résumé:** 13 vulnérabilités fermées sur 14 identifiées. 6 critiques corrigées.
- **Actions Critiques (toutes DONE) :**
  - [x] Sandbox : passage liste noire → liste blanche (ALLOWED_IMPORTS)
  - [x] Sandbox : blocage ast.Name pour open/eval/exec/__builtins__/globals
  - [x] Sandbox : blocage redéfinition noms réservés (def open())
  - [x] Sandbox : PATH whitelist hardcodé, PYTHONNOUSERSITE=1
  - [x] Sandbox : rlimit avec log explicite (plus de silence macOS)
  - [x] EventBus : namespaces protégés system.* / chimera.internal.*
  - [x] EventBus : validation payload (type + taille 256 Ko max)
  - [x] EventBus : rate limit global 500 emit/s
  - [x] EventBus : getRecentEvents() sans payloads (fuite données)
  - [x] WebSocket : rate limit 10 cmd/s par connexion
  - [x] WebSocket : validation longueur commande (5-2000 chars)
  - [x] WebSocket : crypto.timingSafeEqual pour token
  - [x] WebSocket : timeout inactivité 60s + limite message 64 Ko
